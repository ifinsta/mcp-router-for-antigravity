# Documentation

The top level of `docs/` is intentionally small.

Use these files first:

- [`QUICKSTART.md`](./QUICKSTART.md): install, configure, build, and connect the router
- [`DEVELOPMENT.md`](./DEVELOPMENT.md): local development commands and project layout
- [`WINDOWS.md`](./WINDOWS.md): Windows build, packaging, and troubleshooting
- [`BROWSER.md`](./BROWSER.md): browser-backed workflows and performance tooling
- [`API_KEYS.md`](./API_KEYS.md): provider key handling and extension sync notes
- [`CLAUDE_CODE.md`](./CLAUDE_CODE.md): Claude Code MCP setup
- [`INTEGRATIONS.md`](./INTEGRATIONS.md): extension-plus-router architecture

Everything else that is historical, highly specialized, or superseded now lives in
[`archive/`](./archive/README.md).

The product overview remains in [`../README.md`](../README.md).
